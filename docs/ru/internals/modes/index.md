# Режимы сжатия в JSSC

JSSC использует много **режимов сжатия** для достижения оптимального сжатия для разных типов входных данных.

Режим сжатия определяет **как входные данные анализированны, трансформированны и закодированны** в сжатую JavaScript string.

Режимы сжатия — **внутренняя часть алгоритма** и выбираются автоматически во время сжатия.

## Список Режимов Сжатия JSSC
- `00` **No Compression**
- [`01` **Two-Digit CharCode Concatenation**](01)
- [`02` **Two-Byte CharCode Concatenation**](02)
- [`03` **Decimal Integer Packing**](03)
- [`04` **Alphabet Encoding**](04)
- [`05` **Character Encoding**](05)
- [`06` **Inline Integer Encoding**](06)
- [`07` **Frequency Map**](07)
- [`08` **URL**](08)
- `09` **Segmentation** <br>
  > Делит входные данные в независимые сегменты которые сжимаются разными режимами сжатия.
- `10` **String Repetition** <br>
  > Кодирует повторяющуюся string.
- `31` **Recursive Compression** <br>
  > Применяет сжатие рекурсивно к сжатым данным.
